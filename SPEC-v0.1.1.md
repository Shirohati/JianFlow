# 学习待办 v0.1.1 设计规格文档

> **版本**：v0.1.1\
> **日期**：2026-06-02\
> **上一版本**：v0.1.0 (2026-06-01)\
> **变更摘要**：交互补全、嵌套支持、贴附跳转、空状态引导、独立待办挂靠限定、状态操作区细节、每日待办出处跳转、子便签跟随类型变更、回归排序手动选择。

***

## 一、项目概述

### 1.1 定位

一款面向学习场景的桌面待办管理应用，核心理念是**目标驱动日常**——通过便签板管理宏观目标，目标子任务自然下沉到每日待办，形成「规划 → 执行 → 反馈」的闭环。

### 1.2 v0.1.1 核心变更

- **嵌套层级开放**：子便签支持多层嵌套，但跨层级操作受限（不能越级归并）。
- **类型继承明确**：子便签的 `type` 随父便签类型变更自动同步。
- **贴附双向跳转**：画布便签与每日贴附卡片间增加跳转按钮。
- **独立待办挂靠限定**：只能挂靠为子任务，不可变为子便签。
- **回归排序手动选择**：脱离子便签回归时可手动选择在父便签内的插入位置。
- **状态操作区交互**：点击底部小图标展开面板，点击图标或面板外空白处关闭。
- **每日待办出处可跳转**：来源便签名可点击，跳转至目标板并打开对应便签。
- **空状态引导**：画布无便签时显示引导提示。
- **贴附仅作展示**：贴附不改变原便签的任务状态。

## 二、核心概念

### 2.1 便签的双重身份

便签有两种类型，数据结构相同，类型决定显示哪些功能：

| 类型          | 标识 | 作用        | 子任务 | 子便签     | 待办关联 | 截止日期 | 进度条 |
| :---------- | :- | :-------- | :-- | :------ | :--- | :--- | :-- |
| `note` 纯便签  | 📝 | 纯记录，灵感捕捉  | ❌   | ❌       | ❌    | ❌    | ❌   |
| `plan` 阶段计划 | 🎯 | 项目/目标，可拆解 | ✅   | ✅（多层嵌套） | ✅    | ✅    | ✅   |

**关键规则**：

- 切换类型不丢失数据，仅显示/隐藏功能区域。
- note 可转为 plan（通过拖拽到左侧类型区或 ⚙ 菜单），plan 可转回 note（子任务、子便签数据保留，仅隐藏）。
- **子便签的** **`type`** **强制与直接父便签保持一致**：父便签类型变更时，所有后代子便签的 `type` 自动同步（note↔plan 切换），数据结构不变。

### 2.2 染色系统

颜色即分类，拖拽即染色。便签视觉颜色由其所属分类决定。

**两个功能维度**：

- **顶部分类染色区**：改变便签的分类/颜色（工作/学习/阅读/运动/自定义…）
- **左侧类型切换区**：改变便签的类型（note ↔ plan）

### 2.3 状态操作区

便签有四种状态，画布上默认只显示 `active` 状态：

| 状态          | 图标  | 名称  | 含义            | 画布可见 |
| :---------- | :-- | :-- | :------------ | :--- |
| `active`    | —   | 活跃  | 正常使用中         | ✅    |
| `shelved`   | 📦  | 收纳盒 | 搁置/挂起，暂不关注但保留 | ❌    |
| `completed` | 🏆  | 陈列柜 | 完成/归档，展示成就    | ❌    |
| `trashed`   | 🗑️ | 回收站 | 废弃/删除         | ❌    |

所有非 active 状态均可恢复到 active。

### 2.4 子便签、嵌套与越级归并

plan 便签可包含**子便签**。子便签本身也可为 plan 类型（继承父类型），进而继续包含子便签，形成多层级嵌套。

**嵌套规则**：

- 支持任意深度的子便签嵌套（v0.1.x 允许）。
- 子便签的 `type` 始终与其直接父便签相同；若父便签 type 变更，所有后代递归同步。
- **禁止越级归并**：子便签只能挂靠（回归或脱离后重新挂靠）到自己的直接父便签下。不可将第三层子便签直接挂靠到第一层便签，必须先通过中间层。脱离后若想挂靠，只能挂回原直接父级，不能跨级。

**子便签 vs 子任务**：

| 属性       | 子便签 (sub-note)   | 子任务 (sub-task)   |
| :------- | :--------------- | :--------------- |
| 数据结构     | 与顶层便签相同（type 继承） | 简化版（仅标题+状态）      |
| 渲染方式     | 迷你便签（有📌、标题、内容区） | 复选框行             |
| 可拖拽脱离    | ✅                | ❌                |
| 可添加到每日待办 | ✅（子便签的子任务可）      | ✅                |
| 可有自己的子任务 | ✅                | ❌                |
| 可包含子便签   | ✅（若类型为 plan）     | ❌                |
| 内部排序     | ✅（拖拽排序）          | ✅（按 sort\_order） |

**脱离与回归**：

- **脱离**：长按子便签 📌 拖拽到画布空白处，自动设置 `grid_x/grid_y`。
- **回归**：将脱离子便签拖拽至其直接父便签的 📌 热区（高亮），弹出插入位置选择（在该父便签子列表的哪个位置插入：可拖拽到具体排序位置，或通过弹出菜单选择“开头/末尾/某子项前/后”），确认后清除 `grid_x/grid_y` 并更新 `sort_order`。

### 2.5 贴附便签（仅展示关联）

便签可“贴附”到某一天的每日页面，作为快捷参考入口，**不改变便签的任务状态、待办属性**。类似于将便签的快捷方式放到当日。同时建议分区，比如左边的区域是仅粘贴到当日右边是长期粘贴。

## 三、数据模型

### 3.1 TaskItem（统一数据结构）

```
TaskItem
├── id                          唯一标识
├── type                        'note' | 'plan'（子便签该字段自动同步直接父级）
├── sub_type                    'note' | 'task'（仅 parent_id 非空时有效）
├── title                       标题
├── content                     自由文本
├── category_id                 分类 ID
├── priority                    优先级 0/1/2/3
├── parent_id      (nullable)   直接父级 ID，null 为顶层便签或独立待办
├── sort_order                  在同级 parent_id 下的排序
│
├── status                      'active' | 'shelved' | 'completed' | 'trashed'
│
│  ── 画布位置（顶层便签 + 脱离子便签使用）──
├── grid_x        (nullable)    网格列号
├── grid_y        (nullable)    网格行号
│
│  ── 待办相关 ──
├── todo_date     (nullable)    任务执行日（仅子任务/独立待办有效）
├── todo_status   (nullable)    'pending' | 'completed'
├── recurrence    (nullable)    'daily' | 'weekly' | 'monthly' | null
├── completed_at  (nullable)    完成时间
│
│  ── 贴附关联 ──
├── pin_date      (nullable)    贴附展示日，仅用于每日页面快捷展示，不影响待办逻辑
│
├── deadline      (nullable)    截止日期
├── collapsed                    是否折叠
├── note          (nullable)    心得/备注
├── created_at
└── updated_at
```

**关键变更**：增加 `pin_date` 字段，将贴附展示日与原待办执行日 `todo_date` 分离，避免语义冲突。

### 3.2 身份判定规则

| 身份       | 判定条件                                                                 | 说明                    |
| :------- | :------------------------------------------------------------------- | :-------------------- |
| 顶层便签     | `parent_id IS NULL AND status = 'active'`                            | 画布上可见的便签              |
| 子便签（未脱离） | `parent_id IS NOT NULL AND sub_type = 'note' AND grid_x IS NULL`     | 仅在父便签内部显示             |
| 子便签（已脱离） | `parent_id IS NOT NULL AND sub_type = 'note' AND grid_x IS NOT NULL` | 父便签内部 + 画布上同时显示       |
| 子任务      | `parent_id IS NOT NULL AND sub_type = 'task'`                        | 便签内部的复选框项             |
| 独立待办     | `parent_id IS NULL AND todo_date IS NOT NULL AND grid_x IS NULL`     | 在每日页面直接创建             |
| 来自便签的待办  | `parent_id IS NOT NULL AND todo_date IS NOT NULL`                    | 便签子任务被激活到某天           |
| 每日/每周任务  | `recurrence IS NOT NULL`                                             | 重复任务                  |
| 贴附便签     | `pin_date IS NOT NULL`                                               | 在每日页面作为快捷贴附展示，原任务属性不变 |

### 3.3 独立待办挂靠限定

在每日页面创建的独立待办，通过 `[📌 挂靠]` 按钮只能挂靠为某个 plan 便签的**子任务**（`sub_type = 'task'`），不能成为子便签。

**跳转按钮**：

- 待办项右侧的 `[🔗]` 按钮（或直接点击来源便签名），跳转到目标板并打开该便签的浮层。
- 贴附卡片上的 `[打开]` 按钮同样跳转到目标板打开便签。

**贴附便签增加跳转**：

- 画布上便签若已贴附，预览模式显示 📅 小角标，点击该角标可跳转到对应日期的每日页面。

### 4.4 编辑与保存

- **进入编辑**：双击标题或内容，边框高亮。
- **保存**：失焦保存；标题中按 `Enter` 保存；内容中按 `Ctrl+Enter` 保存；显示“保存”按钮。
- **取消**：按 `Esc` 放弃修改，不记录撤销。

### 4.5 撤销/重做

全局历史栈（50 步），支持所有修改操作。`↩ 撤销` `↪ 重做` 按钮及快捷键 `Ctrl+Z` / `Ctrl+Shift+Z`。

## 五、交互设计（精要）

### 5.1 📌 图钉统一操作

同前，长按进入拖拽，打开模式下长按浮层半透明锁定内部交互。

### 5.2 拖拽区域

顶部分类区、左侧类型区、底部状态区均为固定定位，鼠标接近或拖拽时自动完全显现。

### 5.3 回归与排序

脱离子便签拖回直接父便签热区时，父便签子列表展开插入指示线，用户可将便签拖到期望的排序位置（某子项前/后或开头/末尾），松手确认，同时清除网格位置。

### 5.4 嵌套约束

子便签脱离后只能回归到原直接父级，不可跨级挂靠。解除归属后可成为独立顶层便签，但不能再挂靠为任意便签的子便签（除非未来实现“拖入便签内部”通用挂靠，当前版本仅限回归原父级）。

## 六、数据操作补充

### 6.1 贴附

- 每日页面 `[+贴附便签]` 选择便签 → 设置 `pin_date = 当日`。
- 贴附卡片上点击 ✕ → 清除 `pin_date`。
- 画布便签已贴附时显示 📅 角标，点击可跳转至对应日期的每日页面。

### 6.2 子任务跳转

每日待办项旁的 `[🔗]`（或便签名本身）点击 → 导航至目标板，自动打开父便签浮层并滚动至对应子任务。

## 七、状态流转

保持原有状态机，增加贴附字段独立，不影响任务状态。
