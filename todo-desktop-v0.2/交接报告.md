## 三、核心功能清单

### 3.1 目标板 (board.ts)
- 便签渲染：预览模式 / 展开模式 / 节点模式
- 拖拽移动：单选拖拽 / 多选拖拽 / 折叠组拖拽
- 画布操作：鼠标拖动平移 / Ctrl+滚轮缩放 / Ctrl+拖动框选
- 连线系统：SVG贝塞尔曲线 / 端口连接 / 缩放适配
- 分组功能：ComfyUI风格边框 / 折叠为节点 / 双击命名
- 右键菜单：分类染色 / 贴附到每日 / 缩小为节点 / 分区移动
- 全屏编辑：双击内容进入 / ESC退出
- 分区标签：多画布切换 / 右键重命名删除
- 撤销/重做：支持位置/内容/连线/分组/整理操作

### 3.2 每日页面 (home.ts)
- 日期导航 + 小日历快速跳转
- 每日时长显示
- 贴附便签浮动显示
- 右键创建便签 / 转为目标板便签
- 昨日未完成待办自动迁移
- 便签编辑（标题/内容/子任务）

### 3.3 番茄钟 (pomodoro.ts)
- 25分钟工作 / 5分钟休息循环
- 暂停/继续/停止/跳过
- 时间记录自动保存到数据库
- 名言显示

### 3.4 统计报告 (report.ts)
- 时间类型分布 (SVG饼图)
- 月度专注时段 (SVG折线图)
- 日历热力图 (7级热度)
- 每日/周/月/年切换

### 3.5 设置 (settings.ts)
- 主题切换 (4套)
- 画布背景样式
- 分类管理
- 数据导入 (老版本迁移 + 时长导入)
- 数据重置

## 六、已知问题与优化方向

### Critical (必须修复)

| # | 问题 | 说明 |
|---|------|------|
| C1 | board.ts 2200行单文件 | 应拆分为 render/drag/connections/canvas/menus/state 模块 |
| C2 | render()全量DOM重建 | 每次操作都innerHTML替换，50+便签时卡顿。需增量更新 |
| C3 | JSON文件全量写入 | 每次保存都重写整个文件，大数据量下慢且不安全。建议SQLite |
| C4 | 无视图/数据分离 | HTML字符串拼接散布在TS中，难以维护和测试 |

### High (强烈建议)

| # | 问题 | 说明 |
|---|------|------|
| H1 | renderConnections用getBoundingClientRect | 每条连线都触发布局重排，应从数据计算坐标 |
| H2 | boardZCounter无限增长 | 长时间使用后z-index值极大，需定期重置 |
| H3 | initIcons()全局重初始化 | 每次render都处理所有图标，应局部更新 |
| H4 | Store是写缓存 | 数据常直接从API获取而非Store，无失效策略 |
| H5 | CSP已禁用 | tauri.conf.json中csp:null，安全风险 |
| H6 | Google Fonts网络加载 | 离线时字体失败，应本地打包 |
| H7 | 路由不清理页面状态 | 切换页面后番茄钟interval继续运行 |
| H8 | 分组名称仅存内存 | 重启后分组名称丢失，需持久化 |
| H9 | 毛玻璃背景未实现 | settings中有选项但CSS未实现 |
| H10 | 多个设置字段未使用 | bg_home/bg_pomodoro/pomodoro_show_* 等 |

### Medium (建议改进)

| # | 问题 | 说明 |
|---|------|------|
| M1 | 重复工具函数 | icon()/formatMin()/calcProgress()/startEdit() 各页面重复定义 |
| M2 | 前端API无错误处理 | invoke()调用无catch，Rust panic会崩溃前端 |
| M3 | update_task手动字段映射 | 100行if-let，应用serde反序列化替代 |
| M4 | Z-index层级混乱 | 从0到999999无统一规范，需在tokens.css定义 |
| M5 | 缺少键盘快捷键 | 无Ctrl+N创建便签等快捷操作 |
| M6 | 无数据验证 | 前端直接传递Partial\<TaskItem\>无校验 |
| M7 | report数据不刷新 | allRecords加载一次后不更新 |

### Low (可选优化)

| # | 问题 | 说明 |
|---|------|------|
| L1 | modal.ts组件未使用 | 应用使用浏览器原生prompt/confirm |
| L2 | 未使用的CSS变量 | --note-preview-height, --grid-step |
| L3 | build.rs sandbox检测 | 依赖特定IDE环境变量，不够通用 |
| L4 | 连线无唯一性约束 | 可能创建重复连线 |

## 八、构建与部署

### 开发环境
```bash
cd todo-desktop_v0.1
npm install
npm run dev          # 前端开发服务器
npx tauri dev        # Tauri开发模式 (需先启动dev server)
```

### 生产构建
```bash
npm run build        # 构建前端
npx tauri build      # 构建Rust + 打包安装程序
```

输出：
- exe: `src-tauri/target/release/learning_todo.exe`
- 安装包: `src-tauri/target/release/bundle/nsis/笺流_0.1.1_x64-setup.exe`

### 注意事项
- `build.rs` 包含sandbox环境兼容逻辑，通过 `TRAESANDBOX` 环境变量检测
- sandbox下无法执行子进程，tauri-build::build()会panic，使用fallback环境变量
- `Cargo.toml` name保持 `learning_todo`（改名会导致编译问题），显示名通过 `tauri.conf.json` 控制
- 数据目录由 `tauri.conf.json` 的 `identifier` 决定，当前为 `com.learning-todo.desktop`
